# 🪓 Hangman Game (Игра «Виселица»)

Консольная реализация классической игры «Виселица» на чистой Java (без использования сторонних фреймворков). Проект разработан для демонстрации уверенного владения базовым синтаксисом языка, принципами объектно-ориентированного программирования (ООП) и работы со структурами данных.

## 🎯 Игровой процесс
1. Программа случайным образом выбирает слово из встроенного или загружаемого словаря.
2. Игрок видит зашифрованное слово (например, `_ _ _ _ _`) и количество оставшихся попыток.
3. Игрок вводит по одной букве:
   - Если буква есть в слове, она открывается.
   - Если буквы нет, количество попыток уменьшается, а в консоли отрисовывается очередной элемент виселицы (ASCII-графика).
4. Игра продолжается до тех пор, пока игрок не угадает слово (победа) или не закончатся попытки (поражение).

## 🛠 Технический стек и реализованные концепты
- **Java Core (JDK 17+)**
- **ООП (Object-Oriented Programming):** Разделение логики на независимые сущности: `Game` (управление игровым циклом), `Dictionary` (работа со словарем слов), `GallowsPainter` (отрисовка состояний виселицы).
- **Java Collections Framework:** Использование `List` для хранения словаря и `Set` для отслеживания уже введенных игроком букв (чтобы исключить повторные штрафы за одну и ту же ошибку).
- **Регулярные выражения (RegEx) & Валидация:** Проверка пользовательского ввода на корректность (запрет ввода цифр, спецсимволов, пустых строк или нескольких букв за раз).
- **Ресурсы и Стримы (I/O Streams):** Чтение словаря слов из текстового файла.
